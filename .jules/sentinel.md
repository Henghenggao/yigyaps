## 2025-02-28 - Zod URL Validation XSS Risk
**Vulnerability:** Found multiple instances where Zod's `z.string().url()` was used to validate user-provided URLs (like `authorUrl`, `homepageUrl`) in the YigYaps package schema. By default, Zod's `.url()` accepts potentially dangerous protocols such as `javascript:` or `data:`, which could lead to Stored Cross-Site Scripting (XSS) if rendered blindly into `href` tags on the frontend.
**Learning:** Default URL validation in popular libraries like Zod only validates the format of a URL, not its safety for rendering in HTML. Any URI scheme is valid according to the generic URI syntax (RFC 3986), meaning developers must explicitly restrict protocols when validating URLs intended for web links.
**Prevention:** Always combine `z.string().url()` with a `.refine()` check to explicitly allow only safe protocols (e.g., `http://` and `https://`). Ensure the check is case-insensitive, as uppercase protocols (e.g., `JAVASCRIPT:`) are still valid and executable in browsers. Example: `z.string().url().refine((url) => url.toLowerCase().startsWith("http://") || url.toLowerCase().startsWith("https://"))`.
