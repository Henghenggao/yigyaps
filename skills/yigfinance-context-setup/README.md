# Yigfinance: Finance Context Setup

> Mandatory first skill in every Yigfinance analysis chain. Defines analysis boundaries before any computation begins.

## What it does

Finance Context Setup is a structured context-setting protocol that works with you to define:

- **Analysis subject** — entity, business unit, project, or company
- **Reporting basis** — management accounts, IFRS, US GAAP, statutory, etc.
- **Analysis goal** — variance explanation, forecast challenge, investment review, etc.
- **Comparison policy** — FX treatment, adjustment view, scope treatment
- **Business model** — manufacturing, project-based, SaaS, retail, etc.

## Why it matters

Every analysis chain should start here. Without confirmed context:
- Downstream skills may use wrong assumptions
- Reporting bases may be mixed
- Comparability is compromised

## Key features

- **Minimum 3 confirmations** — subject, basis, goal (never skipped)
- **Company profile** — one-time setup, auto-loads for future analyses
- **Session resume** — detects prior analyses and offers to continue
- **Safety-first** — 6 non-negotiable finance safety rules
- **Smart routing** — recommends the right next skill based on your goal

## Skill chain position

```
[Finance Context Setup] → Data Context Check → Variance Review → Management Pack
                        → Data Context Check → Forecast Review → Management Pack
                        → Reporting Basis Check → Plan CFO Review → Management Pack
```
